Stock Management
Angular 6 Node API SQL DB Angular Material Prime NG Gitlab CI/CD

Our client used a pre-existing system, developed in Foxpro, for managing stocks i.e. packing & shipping new stock from warehouse to shops and vice versa. The project was for migrating this legacy system to a Angular web app.

We were a team of two, both freshers and we were given full autonomy over this project. There was little to no knowledge of how the old system worked along with no senior devs in the team.

We had to understand the Foxpro code & reverse engineer it. We initialised by creating scope for project, identifying MVP(minimum viable product). The development was divided into 3 phases and at the end of each phase we would take feedback from client. The system had location and role based authorisation e.g. If you are a junior level employee at a particular location then you would have access to only some particular features.

We created a new UI which had 8 tabs and each tab would mean that stock had reached a particular level. i.e. packed, shipped etc. Each tab also displayed a list of stock in form of table for a selected location. On each level/tab user had the ability to manipulate/update stocks based on their authorisation. We used PrimeNG for tables and Angular Material for designing. Code review was done by a senior dev. There was also manual testing performed. One of major challenges was to make UI responsive as it could also be used in ipad as well.

This was a great learning experience as we were responsible for doing everything, from understanding the system to creating our own stories to creating a new Angular based application. Stakeholders were really happy and they also directly reached to our CTO appreciating our contribution to the project.

Third party integration using SAML
Angular 8 JWT SAML Node API Angular Material

Here I was a part of the team where we had to integrate a third party application using SAML(Security Assertion Markup Language) protocol. It was allowing users to access third party apps using SSO(single sign on). Once user is logged into one app, they can use any app inside of that using SAML assertions.

My contribution to the project was researching on SAML and creating and API which enables to parse the SAML responses to coming to it and if the response is valid, it creates JWT token for user and sends it to our UI. The UI displayed the available leaves of that employee. I also helped in creating interceptors in Angular which were used for handling JWT tokens.

Ionic POC
Ionic Node API SQL DB

Our client uses android devices for scanning and picking stocks from warehouse. They wanted to test Ionic framework for mobile devices. So I was given the opportunity to a POC in Ionic.

They gave the freedom to choose any small feature. So instead of only going for POC we decided to develop a small working prototype which could actually be used by them. My contribution in the project was researching and developing the app in Ionic while identifying how hybrid development was different from native app development and present the conclusions to client.

The app was a simple picking and scanning app. User was able to see a list of stock, which then can be picked and scanned by them. This lasted around a month and based on the POC demo, the client insisted upon continuing the project and create a full fledged app.
